API Documentation

MaLDReTH Research Data Lifecycle API

Available Endpoints

GET /api/stages

Retrieve all lifecycle stages with their metadata.

Response Example:

[ { "id": 1, "name": "Conceptualise", "description": "To formulate the initial research idea...", "position": 0 }, { "id": 2, "name": "Plan", "description": "To establish a structured strategic framework...", "position": 1 } ]

GET /api/tools/{stage_name}

Retrieve all tools associated with a specific stage.

Parameters:

  • stage_name (string): Name of the lifecycle stage

Response Example:

[ { "id": 1, "name": "Miro", "description": "Collaborative whiteboard platform", "url": "https://miro.com", "category": "Mind mapping" }, { "id": 2, "name": "XMind", "description": "Mind mapping software", "url": "https://xmind.net", "category": "Mind mapping" } ]

POST /submit-interaction

Submit user feedback or interaction data.

Request Body:

{ "name": "John Doe", "email": "john@example.com", "organization": "University XYZ", "role": "Researcher", "feedback": "Great visualization!" }

Response Example:

{ "message": "Thank you for your feedback!" }

Error Responses

All endpoints return appropriate HTTP status codes:

  • 200 OK - Request successful
  • 400 Bad Request - Invalid request parameters
  • 404 Not Found - Resource not found
  • 500 Internal Server Error - Server error
